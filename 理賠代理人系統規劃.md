# 理賠代理人內容營銷系統 - 完整規劃

## 現在你的難題
- 看到每天的新聞/案例，想轉化成FB/IG文案教育客戶
- 但手動從新聞 → 法律 → 理賠策略 → 文案，太耗時
- 需要自動化流程：新聞 → 分析 → 文案 → 貼文

## 系統架構

### 模組 1: 新聞 & 法律案例蒐集（自動化）
```
資料來源：
├─ UDN、聯合報（社會版、法律前線）
├─ 勞動部行政判決書（官網爬蟲）
├─ 司法判決書（司法院JIRS）
├─ 勞保署爭議審查決定書
├─ 新聞的留言區（埋藏的真實案例）
└─ 保險消費者申訴（保險局官網）

工具建議：
- 定時爬蟲 (Cron Job) 蒐集上述來源
- 每天自動存入 SQLite 或本地 JSON
- 標籤化：#車禍 #職災 #勞保 #拋棄繼承 #既往症 #失能 等
```

### 模組 2: 智能理賠分析（Claude Code + LLM）
```
輸入: 新聞文本 + 法律段落
處理流程:
1. 抽取關鍵事實：
   - 事件類型（車禍/職災/意外）
   - 當事人身份（被保險人/受益人/債務人）
   - 核心爭議（不理賠理由、既往症、延遲申請）
   - 法院/勞保局判決結果

2. 理賠策略生成：
   - 如果是「既往症」→ 建議因果關係論證
   - 如果是「拋棄繼承逃債」→ 強調勞退不能拋棄的特性
   - 如果是「失能認定」→ 列舉失能等級標準
   - 如果是「調解」→ 提醒還能再申請失能險

3. 客戶教育點：
   - 法律亮點（判決書的1-2句核心論述）
   - 常見誤區（保險公司怎麼用既往症拒賠）
   - 解決方案（應該如何準備資料/申請）
```

## 具體案例：從 UDN 新聞到 FB 文案的完整流程

### 新聞原文（UDN）
```
【標題】亡妹遺產到手姊急拋繼承 銀行追債受阻提告…
【核心】妹妹欠銀行15.6萬，過世2天後姊姊領了74.8萬勞退，
       隨即申請拋棄繼承逃債，法院駁回。
```

### Claude 分析輸出
```
【分析結果】
事件分類: 遺產/繼承爭議 + 勞保理賠 + 債務執行法

爭議點:
1. 拋棄繼承能否逃避被繼承人的債務？
   → 法律：民法1022條，領過遺產就是實際行使繼承
   → 判決：拋棄無效

2. 勞退專戶能否被強制執行？
   → 法律：強制執行法122條，勞退具有優先受償性
   → 判決：此案中勞退已提領，銀行只能向繼承人追償

3. 對理賠代理人的啟示？
   → 既往症爭議也常用「拋棄」逃避責任
   → 應該事先確認：既往症定義、因果關係判定
   → 部份失能也能申請失能險（不只全失能）

【文案方向】
- Hook: 「很多人以為拋棄繼承就能逃債...其實不行」
- 內容: 條文 + 判決邏輯
- CTA: 「你的理賠也可能有法律漏洞，DM我免費檢查」
```

## 你需要的工具棧

### 必需
1. **Claude Code** + mcp_cronjob（定時爬蟲 & 分析）
2. **SQLite/JSON** 存放案例庫
3. **Facebook API** / **Meta Business Suite** API 自動發文

3. **Google Analytics** 追蹤 bio link 點擊（轉化率）
4. **Telegram Bot** 每天早上推播「新案例待分析」
