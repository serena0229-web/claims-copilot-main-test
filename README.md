⚡ 快速開始（3 個步驟）

【第 1 步】閱讀規劃書
  ▶ 打開：理賠代理人系統規劃.md
  ▶ 理解：系統架構、文案模板、預期成效

【第 2 步】初始化系統
  ▶ 打開命令行（或 WSL terminal）
  ▶ 進入這個資料夾：
    cd /mnt/c/Users/user/Desktop/理賠代理人系統
  
  ▶ 初始化：
    node setup.js init
  
  ▶ 會自動建立：
    ~/.claude-insurance-copilot/ 目錄
    ├── config.json
    ├── scripts/
    ├── data/
    └── logs/

【第 3 步】手動測試第一篇新聞
  ▶ 執行：
    node insurance_copilot.js "https://udn.com/news/story/7321/9451800"
  
  ▶ 系統會：
    1. 爬蟲抓取新聞
    2. AI 分析法律爭議
    3. 自動生成 FB 文案
    4. 存入 cases.json

  ▶ 查看結果：
    cat ~/.claude-insurance-copilot/data/cases.json

💡 文案模板（5 種）

1️⃣ 既往症爭議
   Hook: 「你的既往症真的就不賠嗎？」
   ▶ 目標：教育客戶因果關係不同等於理賠

2️⃣ 失能認定
   Hook: 「原來失能險不只賠全失能...」
   ▶ 目標：部分失能也能申請

3️⃣ 拋棄繼承逃債
   Hook: 「拋棄繼承真的能逃債嗎？」
   ▶ 目標：法律漏洞教育（領遺產=實際繼承）

4️⃣ 職災變疾病
   Hook: 「職災後變成疾病，保險公司不理你嗎？」
   ▶ 目標：長期治療也算職災後遺症

5️⃣ 車禍理賠
   Hook: 「車禍全責還能領理賠？」
   ▶ 目標：強制險、公共意外險宣導

❓ 常見問題

Q: 新聞爬蟲會不會違反版權？
A: 只蒐集摘要 + 連結，發文時附來源，改寫成自己的文案。正常新聞引用。

Q: Claude API 費用多少？
A: 分析 1 篇新聞 ≈ $0.01-0.05，100 篇 ≈ $1-5/月。極低成本。

Q: 多久看到成效？
A: 第 2 周開始有互動，第 3 月看到簽約轉化。

Q: 需要人工審核嗎？
A: 第 1-2 周每篇手動審查，第 3 周後 50% 自動 + 50% 待審。

📞 技術支援

【錯誤排查】

1. 新聞爬蟲失敗
   → 檢查網絡：curl -I https://udn.com/news/
   → 查看日誌：tail -100 logs/cron.log
   → 手動測試：node scripts/news_scraper.js

2. Claude API 超額
   → 檢查 API key 在 config.json
   → 查看 Claude Dashboard 使用量
   → 調整分析頻率（隔日分析）

3. 文案質量下降
   → 更新 Claude Prompt
   → 增加人工審核比例
   → 檢查新聞來源品質
